### 3.3 各模块公共 API

| 模块 | 主要导出 | 类型 | 说明 |
|------|---------|------|------|
| `tracked_list` | `TrackedList` | class(list) | 拦截 `__setitem__`/`pop`/`append` 记录步骤快照 |
| `validator` | `validate_sort()` | fn | 排序正确性验证（strict/stalin/stable/topk） |
| | `is_monotonic()` | fn | 单调性检查 |
| | `is_permutation()` | fn | 排列检查（Counter 多集相等） |
| | `check_off_by_one()` | fn | 边界错误诊断 |
| `comparator` | `compare_outputs()` | fn | 输出比对 + 智能差异描述 |
| `sandbox` | `run_code()` | fn | 子进程安全执行（5s 超时） |
| | `run_code_with_collect()` | fn | 执行 + TrackedList 步骤采集 |
| `recognizer` | `recognize_algorithm()` | fn | AST 特征检测 → 算法名（含中文标签） |
| `code_marker` | `mark_code()` | fn | 标记代码行 → `{lines: {outer_loop, compare, swap, ...}, code_lines}` |
| `algo_simulators` | `simulate()` | fn | Python 生成器模拟排序过程 → 步骤列表 |
| `algo_semantic_check` | `check_semantic()` | fn | 白盒语义验证（6 种算法各一检查函数） |
| `step_collector` | `collect_steps()` | fn | 三策略步骤采集（replay / tracked / ast） |
| `feedback` | `diagnose()` | fn | 三层诊断：回溯匹配 → 正则预扫 → AST 静态检查 |
| | `static_check_sort()` | fn | AST 静态结构检查 |
| `grader` | `grade()` | fn | 四维 10 分评分：正确性(4)+算法(2)+质量(2)+效率(2) |
| `complexity` | `estimate()` | fn | 多尺度实测复杂度估算（O(n)/O(n log n)/O(n²)） |
| `similarity` | `similarity()` | fn | AST 令牌余弦相似度（防重命名） |
| | `detect()` | fn | 批量查重（两两比较，超阈值报告） |
| `algo_profiles` | `get_profile()` | fn | 查理论复杂度配置 |
| | `check_ceiling()` | fn | 复杂度门槛检查 |
| `examples` | `EXAMPLE_CODE` | dict | 10 种算法参考实现（冒泡/选择/插入/快速/归并/堆/斯大林/博戈/睡眠/仁慈斯大林） |

### 4.3 中间件与安全

- **CSRF 保护**：`@app.before_request` 对 POST/PUT/DELETE/PATCH 验证 `_csrf_token`（session + form/header 双重校验，`secrets.compare_digest`）
- **认证装饰器**：`@login_required`（检查 session `user_id`）、`@teacher_required`（检查 session `role == 'teacher'`）
- **密码哈希**：`werkzeug.security.generate_password_hash` / `check_password_hash`

## 5. 评测流水线

一次提交（`POST /submit/<id>`）的完整评测流程：

```
用户提交代码
    │
    ▼
[1] ast.parse(code)          ← 共享 AST（全程复用）
    │
    ▼
[2] static_check_sort()      ← 安全检查（eval/exec/__import__ 等黑名单）
    │  └─ 危险调用 → 400 拒绝
    ▼
[3] recognize_algorithm()    ← AST 特征识别算法类型
    │
    ▼
[4] check_semantic()         ← 白盒语义验证（比较方向、循环边界等）
    │
    ▼
[5] mark_code()              ← 标记代码行号（供可视化高亮）
    │
    ▼
[6] 逐用例循环 ────────────────────────────────┐
    │  run_code()           ← 沙箱执行         │
    │  compare_outputs()    ← 输出比对         │
    │  collect_steps()      ← 步骤采集（三种策略）│
    │  写入 SubmissionDetail + SortStep        │
    └──────────────────────────────────────────┘
    │
    ▼
[7] grade()                  ← 四维评分（正确性/算法/质量/效率）
    │
    ▼
[8] diagnose()               ← 智能反馈（三层诊断）
    │
    ▼
[9] db.session.commit()      ← 原子提交
    │
    ▼
返回 JSON（score, grade, feedback, detail_ids, recognized_algo）
```

### 5.1 步骤采集三策略

| 策略 | 函数 | 原理 | highlight_line |
|------|------|------|---------------|
| **C (replay)** | `_collect_replay()` | 识别算法 → 标记代码行 → Python 生成器模拟执行 | ✅ 有（来自 code_marker） |
| **A (tracked)** | `_collect_tracked()` | TrackedList 拦截 `__setitem__`/`pop`/`append` | ❌ None |
| **B (ast)** | `_collect_ast_instrument()` | AST 插桩（预留） | ❌ None |
